# /chronology

1. 读取案件 `matter.md` 获取案件理论和立场。
2. 从用户上传的材料或粘贴的内容中提取事件。
3. 按日期排列，标注重要性和证据支持。
4. 输出时间轴，标注对我方有利/不利的事件，附决策树。

---

# 事实时间轴

## 目的

在中国诉讼中，清晰的事实时间轴是起草起诉状、代理词和备忘录的基础。本技能从案件材料中系统性地提取事件，形成可直接用于法律文书起草的时间轴。

**与美国诉讼的区别：** 中国诉讼无美国式的"chronology"作为证据开示工具。本时间轴是起草和分析工具，不是提交给对方或法院的独立文书（除非作为起诉状事实部分的基础）。

## 立场路由

- `--offensive`（进攻性/原告方）：突出支持我方主张的事件；将对方的不作为标注为违约/侵权节点。
- `--defensive`（防御性/被告方）：突出我方履行义务的事件；标注对方催告/主张的时间节点（用于时效分析和答辩构建）。
- `--neutral`（中立）：不加立场框架，仅按时间排列事件，由律师自行分析。

若未指定标志位，读取 `matter.md` 中的 `role` 字段选择默认值。

## 时间轴格式

```
## 事实时间轴：[案件名称]

立场：[进攻性/防御性/中立]
构建日期：[日期]
材料来源：[已读材料列表]

---

| 日期 | 事件 | 重要性 | 证据支持 | 对案件理论的意义 |
|---|---|---|---|---|
| YYYY-MM-DD | [事件简述] | 🔴 关键 | [证据X，第X页] | [对我方主张的支持/削弱] |
| YYYY-MM-DD | [事件简述] | 🟠 重要 | [合同第X条] | |
| YYYY-MM-DD | [事件简述] | 🟡 相关 | [电子邮件，YYYY-MM-DD] | |
| YYYY-MM-DD | [事件简述] | 🟢 背景 | [口头陈述 — 需书面证据] | |
| YYYY-MM-DD（估算） | [事件简述] | 🟡 相关 | [无直接证据] `[VERIFY: 请核实此日期]` | |

## 关键节点分析

### 诉讼时效分析
- 权利受损时间：[日期]
- 知道或应当知道损害时间：[日期]
- 时效起算：[日期]
- 时效截止：[日期]
- 中断事件：[列表]

### 合同履行节点（若适用）
- 合同签订：[日期]
- 约定履行期限：[日期]
- 实际履行情况：[描述]

### 对我方最有利的事件

### 对我方不利的事件（须在文书中处理）
```

## 提取规则

**从文件提取时：**
- 引号中的日期→事件（合同条款、往来函件）
- 账期/发票/收据的日期
- 电子邮件/微信/短信时间戳
- 官方文件日期（工商、税务、法院等）

**证据质量标注：**
- 书面文件（合同、函件、发票）→强证据，无须额外标注
- 电子证据（未公证的邮件、微信截图）→标注 `[建议公证或通过电子证据保全固定]`
- 口头陈述 →标注 `[仅口头证据——建议补充书面证明]`
- 无直接证据的推断日期 → 标注 `[VERIFY: 请核实]`

## 引用规范

**条文引用：** 同 legal-doc-drafter 的引用规范
**证据引用：** 证据X：[文件名称]（[来源]，[日期]）第[X]页
**法院文书引用：** （[年份]）[法院简称][庭别][字][案号]，第[X]页

## 本技能不做的事

- **凭空生成日期。** 无书面证据支持的日期均标注 `[VERIFY: 请核实]`。
- **对事件的法律意义做最终判断。** 时间轴标注重要性和可能的意义；律师做分析和判断。
- **产出作为证据的文书。** 时间轴是内部分析工具，不是提交给法院或对方的文书。
