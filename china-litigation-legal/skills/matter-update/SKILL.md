# /matter-update

1. 读取指定案件的 `matter.md` 和 `history.md`。
2. 收集用户提供的更新内容。
3. 追加至 `history.md`（最新在上）。
4. 更新 `_log.yaml` 中对应行的 `last_updated`、`next_deadline`、`status`（若有变化）。
5. 若风险等级发生变化，在更新中明确说明变更原因。
6. 确认写入。

# 案件进展更新

## 目的

保持历史记录完整和台账数据最新。每次法庭听证、证据交换、和解接触、外部律师汇报、监管联系或其他重要进展都应记录在案。

## 输入采集

若用户仅提供案件标识而未提供详细内容，询问：

> 请描述本次更新：
> 1. 发生了什么？（事件描述）
> 2. 何时发生的？（日期）
> 3. 下一期限是否有变化？（新的截止日期）
> 4. 风险评估是否需要调整？（是/否/不确定）
> 5. 有新的开放性问题吗？（若有，描述）

## 历史记录条目格式

```markdown
## [YYYY-MM-DD] — [事件类型简述]

[详细描述：发生了什么、谁参与、有什么影响、下一步是什么。
对于庭审：庭审结论、法官/仲裁员反应、下次开庭安排。
对于证据交换：对方提供了什么证据、有无新发现、对案件理论的影响。
对于和解接触：接触方式、对方立场、我方回应、是否继续。
对于外部律师汇报：主要信息、下一步建议、预算状态。]

**下一期限：** [YYYY-MM-DD — 具体事项]
**风险变化：** [维持 [级别] | 从 [X] 升高至 [Y] — [理由] | 从 [X] 降低至 [Y] — [理由]]
```

## 台账更新规则

更新 `_log.yaml` 中对应行的以下字段：
- `last_updated`: [今天日期]
- `next_deadline`: [若有新期限，更新；否则保持原值]
- `status`: [若程序阶段变化，更新，例如从"一审"更新为"二审"]
- `risk`: [若风险等级变化，更新；并在历史记录中说明原因]

## 风险变化的特殊处理

若风险等级发生变化（向上或向下），**必须在历史记录条目中明确说明**：

```markdown
**⚠️ 风险等级变化：**
- 变更前：🟡 中
- 变更后：🟠 高
- 变更原因：[对方新提交的X证据对我方案件理论产生不利影响；…]
- 建议：[立即向合伙人/GC汇报；考虑和解评估]
```

若风险等级从中/低升至高/极高，在更新确认后提示：

> "风险等级已升至[高/极高]。根据实践档案，这可能触发[合伙人/GC]汇报要求。是否需要起草汇报备忘录？运行 `/china-litigation-legal:matter-briefing [标识]` 准备简报。"

## 本技能不做的事

- **评估事件的法律意义。** 它记录用户告知的内容。若用户不确定某事件是否重要，提议运行 `/matter-briefing` 进行深度分析。
- **替代代理律师的工作记录。** 本历史记录是内部追踪，代理律师的正式记录独立维护。
- **自动更改案件状态。** 若案件状态变化（例如一审判决已下，升入二审），提示用户确认后再更新。
