# /customize

读取当前实践档案，询问用户想要更改什么，在档案中更新该项设置，并确认变更。

## 可调整的设置

| 设置 | 位置 | 示例 |
|---|---|---|
| 执业角色 | `## 执业角色` | 律所律师→企业法务 |
| 代理立场 | `## 代理立场` | 原告→两者兼有 |
| 风险偏好 | `## 1. 风险校准 → 风险偏好` | 更新风险容忍度 |
| 严重性阈值 | 风险矩阵 | 调整"高"严重性的金额下限 |
| 处理权限 | 处理权限层级 | 调整升级阈值 |
| 常用法院 | `## 常用法院` | 增加仲裁机构 |
| 升级联系人 | `## 关键内部联系人` | 更新GC/合伙人联系方式 |
| 文书风格 | `## 3. 文书风格` | 更新引用格式偏好 |

## 工作流

1. 询问用户想要更改哪项设置
2. 显示该设置的当前值
3. 采集新值
4. 在档案中更新该项
5. 确认变更

若用户要求大范围重组，建议："这听起来像是需要重做完整设置的范围。运行 `/china-litigation-legal:cold-start-interview --redo` 以进行完整重新访谈。"

## 本技能不做的事

- 删除档案章节。使用 `--redo` 进行完整重置。
- 更改案件层面的数据（matter.md）。使用 `/matter-update`。
