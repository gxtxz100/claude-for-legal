# /matter-close

1. 读取案件 `matter.md` 和 `history.md`。
2. 收集结案信息（结果、最终金额、教训）。
3. 将结案条目追加至 `history.md`。
4. 在 `_log.yaml` 中将案件标注为结案（状态变更为 `结案`）。
5. 可选：将案件文件夹移至归档路径。
6. 确认写入。

---

# 结案归档

## 结案信息采集

| 字段 | 内容 |
|---|---|
| **结案方式** | 判决/和解/调解/撤诉/仲裁裁决/其他 |
| **结案日期** | YYYY-MM-DD |
| **最终结果** | [具体描述：金额、责任认定、禁令等] |
| **最终风险敞口** | [实际损失/支出或获赔金额] |
| **判决/裁决/协议文号** | [文书编号，若有] |
| **履行状态** | [自动履行/需申请执行/已执行完毕] |
| **执行申请期限** | [若有未履行义务，注明2年申请执行期限] |

## 经验教训记录（可选但推荐）

```
## 本案经验教训

**证据方面：**
[什么证据起到了关键作用；什么证据未能采集但本应采集]

**法律策略：**
[什么策略奏效；事后来看有什么不同的做法]

**对方律师/法院：**
[对方的策略特点；法官/仲裁员的倾向]

**操作层面：**
[冲突检查、期限管理、文书准备中的教训]
```

## 结案台账条目

在 `_log.yaml` 中更新对应行：
- `status`: 结案
- `closed_date`: [YYYY-MM-DD]
- `outcome`: [判决胜诉/判决败诉/和解/调解/撤诉/仲裁裁决]
- `final_exposure`: [最终金额]
- `last_updated`: [今天日期]

## 执行期限提醒

若存在未履行义务（对方应当履行但尚未履行），提醒：

> "⚠️ 申请强制执行期限：自生效法律文书确定的履行期限届满之日起**2年内**申请执行，逾期法院不予受理。当前到期日：[YYYY-MM-DD]。建议在此日期前60天在台账中设置提醒。"

## 本技能不做的事

- **删除案件记录。** 仅将案件标注为结案；历史记录完整保留，可供未来参考。
- **自动解除证据保全或法律保存措施。** 若存在相关措施，提醒用户单独处理。
- **评估和解是否合理。** 结案记录的是用户确认的结果；策略判断在此之前发生。
