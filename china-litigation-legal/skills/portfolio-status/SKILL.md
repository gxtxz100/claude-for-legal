# 案件组合状态

## 目的

一次汇总所有活跃案件，让律师一眼看到：有多少案件、风险分布、下周到期什么、哪些诉讼时效临近、哪些案件已经停滞无进展。是合伙人例会的输入材料，也是每日工作清单的基础。

**本周到期：** [N]个期限（最近7天内）

## 异常检测

**台账读取时，自动标注以下异常：**

1. **冲突检查未完成** — `conflicts.status` 不为 `已完成` 或 `已豁免`
2. **诉讼时效临近** — `sof_deadline` 在90天内
3. **诉讼时效已届满** — `sof_deadline` 已过期，案件仍显示为活跃
4. **期限超时未更新** — `next_deadline` 已过但 `status` 仍未更新
5. **停滞案件** — `last_updated` 超过30天
6. **代理律师缺失** — `outside_counsel.firm` 为空，但风险为高或极高
7. **标的额未填写** — `exposure_range` 为空

## 标志位

- `--full` — 完整报告，含所有部分（默认）
- `--deadlines` — 仅显示期限日历（接下来30天）
- `--sof-alerts` — 仅显示诉讼时效预警
- `--stale` — 仅显示停滞案件（超过30天无进展）

## 仪表盘提议

当案件组合超过10个活跃案件时，提议：

> 📊 **以仪表盘形式查看？** 我将构建一个交互式视图，包含：汇总统计（按风险/状态分类）、色码排序表、按类型分布的图表和诉讼时效时间轴。在 Claude Code 中，我将把HTML文件写入 [输出文件夹]，可在浏览器中打开。

## 本技能不做的事

- **评估台账中未记录的案件。** 若案件未通过 `/matter-intake` 录入，汇总中不会出现——漏录比错误数据更安全。
- **更新 `_log.yaml`。** 状态报告是只读的；通过 `/matter-update` 更新案件状态。
- **替代代理律师的进展报告。** 停滞案件的建议是联系代理律师——本插件不猜测外部进展。
